# Contributing

Contributions are welcome! Here's how to get started:

## Process

1. **Fork** this repository
2. **Create a branch** from `main` for your changes
3. **Make your changes** and commit with clear, descriptive messages
4. **Open a pull request** against `main`
5. The repository owner will review your PR

## Guidelines

- Keep changes focused — one feature or fix per PR
- Follow existing code style and conventions
- Include a clear description of what your PR does and why

## What We Won't Accept

- Changes that enable employee monitoring or surveillance (see [USAGE_POLICY.md](USAGE_POLICY.md))
- Breaking changes without discussion in an issue first
